# Paging and Virtual Memory Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add 32-bit paging to BBoeOS, relocate the kernel to a Linux-shaped high-half virtual range (`0xC0000000+`), and run each program in its own per-AS page directory with memory protection from ring 3.

**Architecture:** The kernel binary splits into `boot.bin` (`org 0x7C00`, real-mode + early-PE bootstrap) and `kernel.bin` (`org 0xC0100000`, post-paging high-half). Stage-2 probes E820, loads `kernel.bin` to physical `0x10000`, then early-PE copies it to `0x100000`, builds an initial PD with the first kernel PT identity-mapped + direct-mapped, enables paging, far-jumps high. The high-half kernel allocates 63 more kernel PTs to complete the 256 MB direct map and proceeds with normal init. Programs are loaded into per-program PDs at virtual `0x08048000` (code/BSS) with a 128 KB stack at virtual `0x40000000`.

**Tech Stack:** NASM (flat-binary kernel + program assembly), Python 3 (`cc.py` C compiler, `add_file.py` filesystem tool, pytest test harness), QEMU (`qemu-system-i386`).

**Spec:** [`docs/superpowers/specs/2026-04-28-paging-design.md`](../specs/2026-04-28-paging-design.md)

**Phase boundaries are natural PR boundaries.** Tasks within a phase ship together because the kernel won't boot between intermediate states. Phase boundaries are checkpoints where the kernel is fully functional.

## Status (2026-04-28)

What has shipped pre-Phase-3, in the `worktree-virtual_mem` PR:

- ✅ **Phase 1 Task 1 (BSS trailer widening)** — committed.
- ⏸ **Phase 1 Task 2 (low-memory kernel-data migration)** — *deferred to Phase 3*. The kernel binary at `0x7C00` has no headroom: pulling buffers into kernel BSS makes the binary overlap `SECTOR_BUFFER` at `0xE000`. Once the kernel relocates to physical `0x100000` in Phase 3, this migration becomes trivial.
- ✅ **Phase 2 Task 3 (E820 probe)** — committed.

Net effect on this plan: Phase 3's PR absorbs the deferred migration work and the bitmap allocator alongside the boot-path restructuring. Phase 1 and Phase 2 as standalone PRs are retired in favor of "what's already merged + Phase 3 onwards".

## Phase 1: Toolchain prep (no paging change yet)

After this phase, the kernel still loads at `0x7C00` with no paging. The trailer protocol is widened to support large BSS, and kernel data structures live inside the kernel binary instead of at fixed low-memory addresses. This is a self-contained PR that should produce no user-visible changes.

### Task 1: Widen BSS trailer to 32 bits

**Files:**


Leave `BSS_MAGIC` (`0xB055`) defined for the back-compat read path during migration.

- [ ] **Step 2: Update `cc/codegen/x86/generator.py` to emit the new 6-byte trailer**

Replace the body of `_emit_bss_trailer` (around line 404). The two `self.emit(f"        dw {total}")` and `self.emit("        dw 0B055h")` lines become:

```python

            self.bss_total = total
            self.emit(f"        dd {total}")
        else:
            self.bss_total = "_bss_total_size"
            self.emit("        dd _bss_total_size")
        self.emit("        dw 0B032h")
```

The width changed from `dw` (16-bit) to `dd` (32-bit). The magic is now `0B032h`.

- [ ] **Step 3: Update `bss_setup` in `src/arch/x86/entry.asm` to handle both formats**

Replace the `bss_setup` body (lines 21-48) with logic that accepts both old (4-byte) and new (6-byte) trailers:

```nasm
bss_setup:
        ;; Zero the BSS region of the freshly-loaded program at
        ;; PROGRAM_BASE.  Reads binary size from vfs_found_size, checks
        ;; for either the new 6-byte trailer (dd bss_size; dw BSS_MAGIC32)
        ;; or the old 4-byte trailer (dw bss_size; dw BSS_MAGIC) at the
        ;; end of the binary, then zeroes bss_size bytes immediately
        ;; after.
        push eax
        push ecx
        push edi
        movzx ecx, word [vfs_found_size]
        mov edi, PROGRAM_BASE
        add edi, ecx
        cmp ecx, 6
        jb .try_old_trailer
        cmp word [edi - 2], BSS_MAGIC32
        jne .try_old_trailer
        mov ecx, [edi - 6]                      ; 32-bit BSS size
        sub edi, 6                              ; EDI = start of BSS
        jmp .zero
.try_old_trailer:
        cmp word [edi - 2], BSS_MAGIC
        jne .bss_done
        movzx ecx, word [edi - 4]
        sub edi, 4
.zero:
        test ecx, ecx
        jz .bss_done
        xor eax, eax
        cld
        rep stosb
.bss_done:
        pop edi
        pop ecx
        pop eax
        ret
```

### Task 3: Add E820 probe in stage-2

**Files:**
- Modify: `src/arch/x86/boot/bboeos.asm` (add E820 probe in stage-2 path)

The probe runs in real mode before the PE flip. It walks BIOS INT 15h `AX=0xE820` and writes terminated entries to physical `0x500`.

- [ ] **Step 1: Add E820 probe in `src/arch/x86/boot/bboeos.asm`**

Insert this block in stage-2, after the existing disk reads but before the PE-flip preparation (PIC remap / A20 / GDT). Find the section that does `cli`, A20 enable, and the GDT load — the probe goes immediately before `cli`.

```nasm
        ;; Walk the BIOS memory map via INT 15h AX=E820.  Each entry is
        ;; 24 bytes (8 base, 8 length, 4 type, 4 acpi-attrs).  Stash the
        ;; list at physical 0x500, terminated by a 24-byte zero entry.
        ;; The bitmap allocator (post-paging) reads this to know which
        ;; physical regions are usable.
        mov di, 0x500
        xor ebx, ebx                    ; continuation token, 0 = start
        mov edx, 0x534D4150             ; 'SMAP'
.e820_loop:
        mov eax, 0xE820
        mov ecx, 24
        mov [di + 20], dword 1          ; default ACPI attrs = "valid + ignore-on-read"
        int 0x15
        jc .e820_done                   ; CF set = no support or end
        cmp eax, 0x534D4150
        jne .e820_done
        test ecx, ecx
        jz .e820_skip                   ; zero-length entry, skip
        cmp ecx, 20
        jb .e820_skip
        add di, 24
.e820_skip:
        test ebx, ebx
        jnz .e820_loop
.e820_done:
        ;; Write terminator zero entry at DI.
        xor eax, eax
        mov ecx, 24 / 4
        push di
        rep stosd
        pop di
```

### Task 4: Add bitmap frame allocator

**Files:**
- Create: `src/mm/frame.asm`
- Modify: `src/arch/x86/boot/bboeos.asm` (`%include` the new file)

The allocator is a 256 MB-ceiling bitmap. Functions are callable from kernel code at CPL=0. Initialization is deferred to Phase 3 (after paging is on); for now the file just defines structures and helper functions. We'll wire `frame_init` later.

- [ ] **Step 1: Create `src/mm/frame.asm`**

```nasm
;;; ------------------------------------------------------------------------
;;; mm/frame.asm — physical-frame bitmap allocator.
;;;
;;; Tracks free vs in-use 4 KB physical frames in a static bitmap sized
;;; for a 256 MB ceiling (8 KB bitmap, 32 KB possible bits).  RAM
;;; beyond the ceiling is ignored.  All entry points run at CPL=0.
;;;
;;; Public:
;;;   frame_init(esi = e820_ptr)        scan E820, mark free regions
;;;   frame_alloc()         -> EAX = phys, CF on OOM
;;;   frame_free(eax = phys)            clear the bit
;;;
;;; Reservations: caller invokes frame_reserve_range(EAX = base,
;;; ECX = length) after frame_init to mark BIOS / kernel image / boot
;;; PD + first kernel PT as in-use.
;;; ------------------------------------------------------------------------

%define FRAME_BITMAP_BITS       (256 * 1024 * 1024 / 4096)
%define FRAME_BITMAP_BYTES      (FRAME_BITMAP_BITS / 8)

frame_alloc:
        ;; First-fit scan from frame_search_hint.  Returns EAX = phys
        ;; of allocated frame; CF set on OOM.  Caller zeroes if needed.
        push ebx
        push ecx
        push edi
        mov edi, frame_bitmap
        mov ecx, [frame_search_hint]
        mov ebx, ecx
        shr ebx, 5                              ; dword index
        shl ebx, 2                              ; byte offset
        add edi, ebx
        ;; Bit position within the dword = ECX & 31
        and ecx, 31
.scan_dword:
        mov eax, [edi]
        not eax                                 ; invert: 1 = free
        test eax, eax
        jz .next_dword
        ;; Find lowest set bit at or above ECX.
        bsf eax, eax
        cmp eax, ecx
        jb .next_dword                          ; lowest set bit is below hint
        ;; Found.  Frame number = (edi - frame_bitmap) * 8 + eax
        mov ebx, edi
        sub ebx, frame_bitmap
        shl ebx, 3                              ; bytes -> bits
        add eax, ebx
        ;; Out-of-range check.
        cmp eax, FRAME_BITMAP_BITS
        jae .oom
        ;; Mark the bit set in the bitmap.
        mov ecx, eax
        mov ebx, 1
        shl ebx, cl
        and ecx, ~31
        shr ecx, 3                              ; byte offset
        or [frame_bitmap + ecx], ebx
        ;; Update hint and free count.
        mov [frame_search_hint], eax
        dec dword [frame_free]
        ;; Convert frame number to physical address.
        shl eax, 12
        clc
        pop edi
        pop ecx
        pop ebx
        ret
.next_dword:
        add edi, 4
        xor ecx, ecx                            ; only the first dword respects the hint
        cmp edi, frame_bitmap + FRAME_BITMAP_BYTES
        jb .scan_dword
.oom:
        stc
        pop edi
        pop ecx
        pop ebx
        ret

frame_free:
        ;; EAX = phys.  Clears the bit, updates hint if EAX < hint.
        push ebx
        push ecx
        shr eax, 12                             ; frame number
        cmp eax, FRAME_BITMAP_BITS
        jae .out_of_range
        mov ecx, eax
        and ecx, 31
        mov ebx, 1
        shl ebx, cl
        not ebx
        mov ecx, eax
        shr ecx, 3
        and ecx, ~3
        and [frame_bitmap + ecx], ebx
        inc dword [frame_free]
        cmp eax, [frame_search_hint]
        jae .done
        mov [frame_search_hint], eax
.done:
        pop ecx
        pop ebx
        ret
.out_of_range:
        ;; Silently ignore — frees of out-of-range addresses are no-ops.
        pop ecx
        pop ebx
        ret

frame_init:
        ;; ESI = pointer to E820 list at 0x500 (now reachable as
        ;; 0xC0000500 via direct map; caller fixes up).  Walks until a
        ;; zero-length entry, marks type=1 regions as free, leaves
        ;; everything else reserved.
        push eax
        push ebx
        push ecx
        push edx
        push edi
        ;; Start from all-reserved.
        mov edi, frame_bitmap
        mov ecx, FRAME_BITMAP_BYTES / 4
        mov eax, 0xFFFFFFFF
        cld
        rep stosd
        mov dword [frame_free], 0
.entry_loop:
        mov eax, [esi + 0]                      ; base low
        mov ebx, [esi + 8]                      ; length low
        mov edx, [esi + 16]                     ; type
        ;; Zero-length entry = end of list.
        test ebx, ebx
        jnz .got_entry
        cmp dword [esi + 12], 0
        je .done
.got_entry:
        cmp edx, 1                              ; type 1 = usable
        jne .skip
        call frame_mark_range_free              ; (eax = base, ebx = length)
.skip:
        add esi, 24
        jmp .entry_loop
.done:
        ;; frame_total = bits we touched as free.  For now, leave it
        ;; equal to frame_free; caller updates frame_total separately.
        mov eax, [frame_free]
        mov [frame_total], eax
        mov dword [frame_search_hint], 0
        pop edi
        pop edx
        pop ecx
        pop ebx
        pop eax
        ret

frame_mark_range_free:
        ;; EAX = base, EBX = length.  Marks complete frames within the
        ;; range as free.  Skips partial frames at either end.
        push eax
        push ebx
        push ecx
        push edx
        ;; Round base up to frame boundary.
        add eax, 0xFFF
        and eax, ~0xFFF
        ;; Round (base + length) down to frame boundary.
        ;; Recompute end from original base.
        mov ecx, [esp + 12]                     ; original base
        add ecx, ebx                            ; end
        and ecx, ~0xFFF
        cmp eax, ecx
        jae .done
.loop:
        push eax
        push ecx
        shr eax, 12                             ; frame number
        cmp eax, FRAME_BITMAP_BITS
        jae .skip
        mov ecx, eax
        and ecx, 31
        mov ebx, 1
        shl ebx, cl
        not ebx
        mov ecx, eax
        shr ecx, 3
        and ecx, ~3
        and [frame_bitmap + ecx], ebx
        inc dword [frame_free]
.skip:
        pop ecx
        pop eax
        add eax, 0x1000
        cmp eax, ecx
        jb .loop
.done:
        pop edx
        pop ecx
        pop ebx
        pop eax
        ret

frame_reserve_range:
        ;; EAX = base, ECX = length.  Marks complete frames within the
        ;; range as reserved (in-use).
        push eax
        push ebx
        push ecx
        push edx
        ;; Round base down to frame boundary.
        and eax, ~0xFFF
        ;; Round (base + length) up.
        add ecx, eax                            ; end
        add ecx, 0xFFF
        and ecx, ~0xFFF
.loop:
        cmp eax, ecx
        jae .done
        push eax
        push ecx
        shr eax, 12
        cmp eax, FRAME_BITMAP_BITS
        jae .skip
        mov ecx, eax
        and ecx, 31
        mov ebx, 1
        shl ebx, cl
        mov edx, eax
        shr edx, 3
        and edx, ~3
        ;; Only decrement free count if the bit was clear.
        test [frame_bitmap + edx], ebx
        jnz .already_set
        or [frame_bitmap + edx], ebx
        dec dword [frame_free]
.already_set:
.skip:
        pop ecx
        pop eax
        add eax, 0x1000
        jmp .loop
.done:
        pop edx
        pop ecx
        pop ebx
        pop eax
        ret

        section .bss
        align 4
frame_bitmap:
        resb FRAME_BITMAP_BYTES
frame_total:
        resd 1
frame_free:
        resd 1
frame_search_hint:
        resd 1
        section .text
```

NASM flat-binary output doesn't actually use `.bss` as a section, so adapt the BSS reservation as a plain `times` block at the end (NASM convention used elsewhere in BBoeOS):

```nasm
        align 4
frame_bitmap:
        times FRAME_BITMAP_BYTES db 0
frame_total:    dd 0
frame_free:     dd 0
frame_search_hint: dd 0
```

(Replace the section blocks above with this plain layout — it matches existing kernel asm conventions.)

## Phase 3: Boot path restructuring and paging enable

This phase ships as one PR — the kernel doesn't boot in any intermediate state.

After this phase: kernel binary splits into `boot.bin` + `kernel.bin`, kernel runs at virtual `0xC0100000`, paging is on, the 256 MB direct map is wired up, and `kernel_pd_template` is the PD in CR3. Programs *still load and run at the old physical addresses* — that comes in Phase 4. So at the end of this phase we have paging on but every program shares `kernel_pd_template`, which doesn't yet have any user-half mappings — meaning programs won't actually run yet.

To keep this phase shippable on its own, **Task 9 includes a temporary "shim" that gives `kernel_pd_template` user-mappings sufficient to run programs at the old `PROGRAM_BASE = 0x600` virtual layout, mapped to private user frames.** This is a one-PD-shared-by-everyone interim state. Phase 4 replaces it with per-program PDs.

```nasm
        ;; Reload segment registers with kernel data selector.
        mov ax, 0x10
        mov ds, ax
        mov es, ax
        mov ss, ax
        mov fs, ax
        mov gs, ax
```

### Task 8: High-half kernel entry — drop identity, init bitmap, allocate remaining PTs

**Files:**
- Modify: `src/arch/x86/kernel.asm` (`high_entry`)

- [ ] **Step 1: Replace `high_entry` with the full high-half boot**

```nasm
high_entry:
        ;; Reload segment registers (CS already loaded by far-jump).
        mov ax, 0x10
        mov ds, ax
        mov es, ax
        mov ss, ax
        mov fs, ax
        mov gs, ax
        mov esp, kernel_stack_top               ; kernel-virt now

        ;; Drop the identity mapping at PDE[0].  After this, virtual
        ;; addresses in 0..0x3FFFFF are unmapped.  We've already
        ;; jumped high so EIP is in 0xC0xxxxxx.
        mov eax, [boot_pd_phys]
        ;; boot_pd_phys is a constant 0x1000; we could just hardcode it.
        ;; But for clarity, use a kernel-virt alias.
        mov dword [0xC0000000 + 1000h], 0
        ;; Flush TLB by reloading CR3.
        mov eax, cr3
        mov cr3, eax

        ;; Promote boot PD into kernel_pd_template by recording its
        ;; physical address.
        mov dword [kernel_pd_template_phys], 0x1000

        ;; Initialize bitmap allocator from E820 at virtual 0xC0000500.
        mov esi, 0xC0000500
        call frame_init

        ;; Reserve regions used at boot:
        ;; - VGA / BIOS at 0xA0000..0xFFFFF
        ;; - Kernel image at 0x100000..(__pa(kernel_end))
        ;; - Boot PD at 0x1000
        ;; - First kernel PT at 0x2000
        mov eax, 0xA0000
        mov ecx, 0x60000                        ; 0xA0000..0xFFFFF inclusive
        call frame_reserve_range

        mov eax, 0x100000
        mov ecx, kernel_end - 0xC0000000 - 0x100000     ; physical kernel size
        call frame_reserve_range

        mov eax, 0x1000
        mov ecx, 0x1000
        call frame_reserve_range

        mov eax, 0x2000
        mov ecx, 0x1000
        call frame_reserve_range

        ;; Allocate the remaining 63 kernel PTs (one per 4 MB chunk
        ;; from physical 0x400000 to 0x10000000), populate each as a
        ;; direct-map PT, install in kernel_pd_template's PDEs[769..831].
        mov ebx, 769                            ; first PDE index
.alloc_kernel_pt:
        call frame_alloc
        jc .panic                               ; OOM during boot — panic
        ;; EAX = phys of new PT (within first 4 MB, reachable via
        ;; existing direct map at PDE[768]).
        mov edi, eax
        add edi, 0xC0000000                     ; kernel-virt address of the new PT
        push eax                                ; save phys for PDE
        ;; Populate PT: PTE[j] = (pde_index - 768) * 0x400000 + j * 0x1000 | P | RW | G
        mov ecx, ebx
        sub ecx, 768                            ; chunk index 1..63
        shl ecx, 22                             ; chunk_index * 4 MB
        xor edx, edx
.pt_fill:
        mov eax, ecx
        add eax, edx
        or eax, 0x103                           ; P | RW | G
        mov [edi + edx], eax
        ;; Move to next 4 KB step.
        add edx, 0x1000 / 4 * 4                 ; +4 bytes per PTE
        cmp edx, 1024 * 4
        jb .pt_fill

        ;; Install in kernel_pd_template at PDE[ebx].
        pop eax                                 ; phys of new PT
        or eax, 0x003                           ; P | RW
        mov edi, 0xC0000000 + 0x1000            ; kernel_pd_template
        mov [edi + ebx*4], eax

        inc ebx
        cmp ebx, 832
        jb .alloc_kernel_pt

        ;; Flush TLB.
        mov eax, cr3
        mov cr3, eax

        ;; Continue with the existing post-flip init: TSS, IDT, drivers,
        ;; VFS, NIC, shell.  This is the body of `protected_mode_entry`
        ;; from `entry.asm`.
        jmp protected_mode_entry

.panic:
        ;; Couldn't allocate kernel PTs — print marker and halt.
        mov al, '!'
        out 0xE9, al
        cli
        hlt
        jmp $ - 1
```

- [ ] **Step 1: Create `src/mm/as.asm` per the spec's API contract**

```nasm
;;; ------------------------------------------------------------------------
;;; mm/as.asm — per-program address-space helpers.
;;;
;;; Builds and tears down user PDs.  Kernel-half PDEs (768..1023) are
;;; copied verbatim from kernel_pd_template at as_create time and
;;; never modified afterward.
;;;
;;; Public:
;;;   as_create()                    -> EAX = pd_phys, CF on OOM
;;;   as_destroy(eax = pd_phys)      free user PTs and pages, then PD
;;;   as_map_page(eax pd, ebx virt,
;;;               ecx phys, edx flags) -> CF on OOM
;;;   as_unmap_page(eax pd, ebx virt) clear PTE; invlpg if active
;;; ------------------------------------------------------------------------

as_create:
        push ebx
        push ecx
        push edi
        push esi
        call frame_alloc
        jc .oom
        ;; Zero the new PD via direct map.
        mov edi, eax
        add edi, 0xC0000000
        push eax                                ; save pd_phys
        mov ecx, 1024
        xor ebx, ebx
.zero:
        mov [edi + ebx*4], ebx
        inc ebx
        cmp ebx, ecx
        jb .zero
        ;; Copy top-256 PDEs from kernel_pd_template.
        mov esi, [kernel_pd_template_phys]
        add esi, 0xC0000000
        add esi, 768 * 4
        add edi, 768 * 4
        mov ecx, 256
        cld
        rep movsd
        pop eax
        clc
        pop esi
        pop edi
        pop ecx
        pop ebx
        ret
.oom:
        stc
        pop esi
        pop edi
        pop ecx
        pop ebx
        ret

as_destroy:
        ;; Walks user-half PDEs (0..767), frees each PT's user-page
        ;; frames, frees PT frames, then frees the PD frame itself.
        push eax
        push ebx
        push ecx
        push edx
        push edi
        push esi
        mov esi, eax                            ; pd_phys
        mov edi, eax
        add edi, 0xC0000000                     ; pd direct-map address
        xor ebx, ebx                            ; PDE index
.pde_loop:
        mov eax, [edi + ebx*4]
        test eax, 1                             ; P bit
        jz .next_pde
        ;; PDE present.  Walk the PT.
        mov edx, eax
        and edx, 0xFFFFF000                     ; PT phys
        push edx
        add edx, 0xC0000000                     ; PT direct-map address
        xor ecx, ecx
.pte_loop:
        mov eax, [edx + ecx*4]
        test eax, 1
        jz .next_pte
        and eax, 0xFFFFF000                     ; user-page phys
        call frame_free
.next_pte:
        inc ecx
        cmp ecx, 1024
        jb .pte_loop
        ;; Free the PT frame.
        pop eax                                 ; PT phys
        call frame_free
.next_pde:
        inc ebx
        cmp ebx, 768
        jb .pde_loop
        ;; Free the PD itself.
        mov eax, esi
        call frame_free
        pop esi
        pop edi
        pop edx
        pop ecx
        pop ebx
        pop eax
        ret

as_map_page:
        ;; EAX = pd_phys, EBX = user_virt, ECX = phys, EDX = flags.
        ;; Inserts a PTE.  Allocates a PT frame on demand if PDE is
        ;; not-present.  Does not invalidate TLB.
        push eax
        push ebx
        push ecx
        push edx
        push edi
        push esi
        ;; Compute PDE index = virt >> 22.
        mov esi, ebx
        shr esi, 22
        ;; Read PDE.
        mov edi, eax
        add edi, 0xC0000000
        mov eax, [edi + esi*4]
        test eax, 1
        jnz .pde_present
        ;; Allocate a fresh PT.
        push edx
        push esi
        push edi
        call frame_alloc
        jc .oom_no_pop
        push eax
        ;; Zero the new PT.
        mov edi, eax
        add edi, 0xC0000000
        mov ecx, 1024
        xor edx, edx
.pt_zero:
        mov [edi], edx
        add edi, 4
        loop .pt_zero
        pop eax
        ;; Install PDE = pt_phys | P | RW | U.
        or eax, 0x007
        pop edi
        pop esi
        mov [edi + esi*4], eax
        and eax, 0xFFFFF000                     ; PT phys
        pop edx
.pde_present:
        ;; PT phys in EAX (low bits possibly set; mask).
        and eax, 0xFFFFF000
        add eax, 0xC0000000                     ; PT direct-map
        ;; Compute PTE index = (virt >> 12) & 0x3FF.
        mov ebx, [esp + 16]                     ; original user_virt
        shr ebx, 12
        and ebx, 0x3FF
        ;; Build PTE = phys | flags.
        mov ecx, [esp + 12]                     ; original phys
        and ecx, 0xFFFFF000
        or ecx, [esp + 8]                       ; flags
        mov [eax + ebx*4], ecx
        clc
        pop esi
        pop edi
        pop edx
        pop ecx
        pop ebx
        pop eax
        ret
.oom_no_pop:

        pop ebx
```

## Notes for the executing engineer

- The boot path (Phase 3) is genuinely subtle — the kernel can't run between "split files" and "paging on + far-jump high". That's why Phase 3's tasks land as one PR and the temporary shim in Task 9 keeps programs runnable until Phase 4 replaces it.
- When debugging boot-time faults, the only diagnostic is `EXCnn` on serial. Use `qemu-system-i386 ... -serial stdio` and watch the output. Common mistakes: forgetting to flush TLB after PDE updates (CR3 reload or `invlpg`), forgetting the WP bit so kernel writes silently bypass R/W=0 PTEs, sign-extension issues on 16-bit values widened to 32-bit.
- The frame allocator's bitmap arithmetic (`bsf`, byte/dword indexing, hint updates) is easy to get subtly wrong. If allocations succeed but two callers receive the same physical address, the bitmap update is racy or incorrect.
- `as_destroy`'s walk must handle PDEs marked present but pointing at a PT with all-zero PTEs — the PT frame still needs to be freed even if no user pages are present.
- The kernel BSS scratch buffer (`program_scratch`, 256 KB) is sized for the largest expected program. If a program grows past 256 KB, growing the buffer is a one-line change.
- Tests in `tests/test_asm.py` exercise the self-hosted assembler against every `static/*.asm`. If post-paging the assembler produces wrong output, the bug is likely in `program_enter`'s page mapping (a partial last page, or BSS accidentally not zeroed).
